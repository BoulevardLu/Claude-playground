Context / 上下文

研究背景与核心问题

金融交易平台的页面布局直接影响用户的决策效率与操作体验。「横向资产选择器 + 下方内容区展开」是外汇、加密货币、股票行情类产品中最常见的 Layout 模式，但不同产品在信息密度、操作入口与资讯布局上存在显著差异。

本报告解决的核心问题是：顶级交易平台如何组织页面空间？在资产切换、图表展示、操作面板和资讯模块的布局上有哪些共性与差异？通过拆解 Kraken、Stocktwits、Robinhood、Bloomberg 四个案例，并对标万里汇「安心兑」现有设计，我们旨在沉淀一份可落地的 Layout 优化策略。

整体 Layout 模式

交易型页面的三区架构

通过对多个交易平台的分析，这类页面的空间分配高度一致：顶部横向资产选择器锁定「关注标的」，页面主体分为左侧导航/Watchlist、中央内容区、右侧操作/资讯栏三个纵向区域。万里汇「安心兑」同属此架构，以下先建立模型再逐一对照真实案例。

交互与架构分析

Kraken Markets 的架构策略是「先总览，再深入」：页面主体是一张全宽数据表，陈列所有资产的实时报价、涨跌幅、交易量，用户通过行点击进入资产详情页（跳页）或直接点击行内「Buy」进入交易流程。

  • 顶部 Ticker 带是展示型，不承担「选择标的」的职责，仅提供环境数据——这是与万里汇最大的差异
  • 收藏系统（⭐）让用户在大表中优先置顶自己关注的资产，承担了万里汇「我收藏的」Tab 的相同职责
  • 「Assets (3)」Tab 上的数量 Badge 让用户在切换前就知道当前筛选条件下有多少资产
  • 表格行内放置「Buy / Sell」按钮，把操作入口前置到列表层——适合高频交易用户，但对需要先「看清楚再决策」的汇兑场景压力较大
  • 新闻/资讯完全不在此页，操作前的信息准备环节依赖外部跳转

视觉设计分析

  • 双灰度交替行（#FFF / #FAFAFA）是金融数据表最经济的行区分方案，视觉成本极低
  • 红/绿仅用于涨跌数字，整张表格除此之外全为中性灰，语义映射精确
  • 等宽数字字体保证所有价格列对齐，扫读效率高
  • 列宽按「信息重要性」分配：资产名称列最宽，价格列次之，辅助列（Volume）更窄
  • 顶部 Ticker 带有意降权（浅灰底，小字），避免与主表竞争视觉注意力

交互与架构分析

这是与万里汇「安心兑」架构最接近的案例，完整实现了三列布局：左侧 Watchlist、中央详情、右侧资讯。中央区从上到下依次是：

  • 第一层：大价格 + 关键指标横排卡片（Market Cap / Volume / 52W High / Low / PE）——建立「选中标的的上下文认知」，等同于万里汇的汇率大字 + 涨跌幅
  • 第二层：Sub-Tab（About / Feed / News / Sentiment / Earnings / Fundamentals）——切换内容维度，与万里汇「概览 / 市场资讯 / 财经日历 / 汇率提醒 / 进行中订单」完全同构
  • 第三层：图表——K 线 + 时间轴控制条，交互逻辑高度一致
  • CTA「Trade」放在资产名称行的最右侧，始终可见但不干扰阅读流
  • 右侧新闻栏独立存在，不随 Sub-Tab 切换而变化——始终展示「Latest NVDA News」。这使新闻成为页面「稳定的背景情报」，而非 B 区的「按需加载内容」
  • 相比万里汇：万里汇把「市场资讯」放入 Sub-Tab 内部（隐藏在 Tab 后），Stocktwits 把新闻永久置于右侧——哪种更好，取决于资讯对于决策的重要程度

视觉设计分析

  • 绿色 Trade 按钮与绿色涨幅文字使用同色系，形成视觉暗示「上涨时买入」的情感引导
  • 指标卡片（Market Cap 等）用细边框白底小卡，从主背景浮出，视觉权重中等——不喧宾夺主但也足够显眼
  • Sub-Tab 选中态仅靠红色下划线区分，字体加粗，完全不填充背景色，页面整体色彩保持克制
  • 右侧新闻栏用分割线区隔每条新闻，密度高但不凌乱；缩略图（Thumbnail）辅助快速识别来源
  • 大价格字号（≈34px）→ 涨跌（16px 绿）→ 指标卡片（12px）三级字号阶梯清晰

交互与架构分析

同一页面在深色模式下展示了「Feed」Tab 内容，即交易社区的帖子流。这让我们可以看到 B 区 Sub-Tab 展开后的真实内容密度。

  • K 线图相比浅色版信息密度提升：增加了成交量柱（绿/红），配合时间轴（7/30 到 8/25），用户可直观看到价格与成交量的关联
  • 时间范围选择条（1D / 5D / 1M / 3M / 1Y / 3Y）放在图表正下方，与图表强绑定，和万里汇的「1D / 7D / 14D / 1M」完全同构
  • Feed 内容中的「Bullish / Bearish」情绪 Badge是 Stocktwits 独有的社区情绪量化，值得万里汇借鉴——可以类比「市场多空情绪指数」或「相似走势对比」
  • 深色模式下，K 线图的绿/红对比更加鲜明，适合长时间盯盘使用场景
  • 顶部资产 Header（NVDA + 价格 + Trade 按钮）在深色和浅色两版中设计完全一致——说明这是该 Layout 中优先级最高、最稳定的核心区域

视觉设计分析

  • 深色系（#121212）大幅提升图表的数据可读性：红/绿蜡烛在黑底上的对比远强于白底，尤其适合长时间专注阅读
  • 价格文字用亮绿色（#39d353）显示上涨，视觉冲击力在深色背景下进一步增强，情感上暗示「机会」
  • Watchlist 侧栏的 Sparkline 小图在深色模式下同样清晰，绿/红单色折线即可传递趋势
  • 右侧新闻栏在深色背景下用细分割线（#2a2a2a）分隔，轻盈不压沉
  • Sub-Tab 选中态（Feed）用白色 Underline，比浅色模式的红色更低调，避免与图表的彩色数据竞争

交互与架构分析

Robinhood Web 版采用了与移动端风格迥异的「图表全宽优先」策略：整个内容区让给了 K 线图和技术指标，取消了右侧新闻栏，将 CTA「Trade TSLA」固定在顶部 Nav 最右侧。

  • CTA 策略：顶部固定——无论用户在图表区如何缩放和滚动，Trade 按钮始终在视口最右侧，触达成本为零
  • 顶部 Nav 的构成：资产价格（左）+ 搜索栏（中）+ 操作按钮（右），形成「上下文 → 探索 → 行动」的水平信息流
  • 图表全宽布局给技术派用户提供了充分的「分析空间」，左侧竖列指标图例（MA / EMA / RSI 等）可以独立控制开关
  • 缺点：没有新闻和基本面信息，用户需要跳出当前页到外部获取这些上下文信息。这对汇兑场景不适用——外汇市场高度受宏观新闻影响，资讯必须在决策环节可见
  • 资讯在本设计中完全缺位，是 Robinhood 面向「已经做好决策」的高频交易用户的刻意取舍

视觉设计分析

  • 橙色（#FF6600）CTA 在全黑页面中是唯一暖色，跳出率极高，但也可能让人误解为「警告信号」——颜色语义在金融语境中需谨慎
  • 多指标叠加（橙 / 蓝 / 绿 / 红线）在同一图表上，需要严格的图例说明，否则认知负担巨大
  • 深色系（#121212）+ 彩色指标线，是专业分析工具的标准视觉语言
  • 顶部价格数字（白色大字）与红色负变化（-$143.49）对比强烈，亏损状态一目了然
  • 单色折线（无蜡烛图）使图表更简洁，适合快速判断趋势而非详细分析

交互与架构分析

Bloomberg 是媒体驱动的金融平台，没有直接的交易入口，但它是「资讯如何与图表协同布局」的最佳参考案例。页面分三列：左侧是「实时行情 + 图表」，中央是「深度数据可视化（热力图）」，右侧是「新闻标题流」。

  • 右侧新闻流的价值：以标题 + 时间戳 + 小缩略图的形式排列，信息密度高，可快速扫读 10+ 条资讯，绝不打断左侧图表的阅读
  • 「突发新闻 Banner」叠加在图表上方，以红色横幅形式出现——这是紧急市场事件打断正常浏览流的交互设计，值得在汇率市场的「重大波动预警」场景中借鉴
  • 热力图（红绿色矩阵）是将「时间维度」可视化的有效工具——万里汇的「技术指标」区块可参考此方式将「价格分位」等多维数据展示出来
  • 顶部二级 Nav（Live Now / Markets / …）是内容分类入口，功能上等同于万里汇的 Sub-Tab，但以全局 Nav 的形式呈现，优先级更高
  • Bloomberg 没有 CTA，但其「Subscribe」按钮固定在顶部右侧——这印证了所有金融类产品最重要的操作入口都锚定在顶部右侧区域这一设计规律

视觉设计分析

  • 全黑底（#000）+ 白字，配合红色 Breaking News Banner，形成「紧迫感」的视觉语言——适合新闻媒体，但对汇兑平台可能过于压迫
  • 热力图的红绿渐变色阶是色彩编码的典型应用：颜色即数据，无需图例也能直觉理解
  • 新闻流右侧栏的字号（≈12px）和行间距经过精心调整，在 20-25% 的栏宽内仍保证可读性
  • 双层 Nav（品牌层 + 内容分类层）清晰区分了「身份认知」和「功能导航」，层级不混淆
  • 图表和新闻之间没有明显分割线，仅靠视觉重量和内容类型的差异来建立区隔——留白即分隔
万里汇设计评估

「安心兑」页面架构的三区分析与优化方向

基于上述 5 个案例，横向对照万里汇现有方案，识别已做到位的决策与仍可优化的空间。

万里汇「安心兑」— 当前 Layout 架构评估

三区架构（左导航 A + 中央内容 B + 右侧操作面板 C）已与行业主流对齐，以下对每区做细化分析

A 区 — 左侧主导航

万里汇用固定左侧导航栏承担全局功能入口（首页 / 资金管理 / 账单中心等），这是典型的 SaaS 产品布局，与纯投资类平台（Watchlist 侧栏）有所不同。


当前做法：导航与内容区完全分离，各自独立，清晰。


可优化方向：考虑在「安心兑」功能内为左侧栏增加「当前关注货币对」的快速跳转列表（参考 Stocktwits Watchlist），减少频繁使用时的顶部列表滚动成本。

B 区 — 中央内容区

顶部横向货币对列表 → 下方汇率大数字 + 图表 + 5 个 Sub-Tab，架构完整且与行业标杆（Stocktwits）高度对齐。


已做到位：Sub-Tab 数量合理（5 个），图表 + 时间维度控制条联动，市场资讯 AI 摘要是差异化亮点。


可优化方向：在大价格与图表之间补充「横向关键指标行」（近 7 日高/低、历史分位），让用户在看图表前先建立数值锚点，减少认知跳跃。

C 区 — 右侧操作面板

「立即兑换 / 指定汇率」Tab 式面板固定于右侧，不随主内容滚动，策略正确且执行完整。


已做到位：Sticky 固定、双模式切换（立即/指定）、面板底部的「短期锁汇 / 远期锁汇」附加产品引导，形成完整的产品生态闭环。


可优化方向：右侧面板目前不含任何资讯信息。可参考 Stocktwits 右侧新闻栏的做法，在操作面板下方增加「与此货币对相关的近 3 条快讯」，让用户在填写金额时仍能感知市场状态。

顶部横向列表 — 当前优势

货币对卡片设计包含国旗 + 货币对名称 + 汇率 + 涨跌幅 + 收藏星，信息密度适当。「我收藏的 / 最近浏览的」分类 Tab 是比单纯收藏更高级的 Watchlist 设计——这是万里汇超越 Kraken / Stocktwits 的地方。

建议保留并强化：增加选中卡片的视觉连接（如「▼」指示箭头或卡片下边框高亮延伸至主内容区），明确告知用户「当前展示内容对应哪个卡片」。

资讯信息的布局策略建议

对比 5 个案例，新闻/资讯有两种主流布局策略：

① Sub-Tab 内置（当前万里汇做法）：按需展示，不占额外空间，但需要用户主动点击才能获取，且默认状态下感知不到「有新资讯」。

② 右侧固定栏（Stocktwits / Bloomberg）：始终可见，被动接受，适合「资讯对决策至关重要」的场景。建议在「概览」Tab 状态下，将最新 2-3 条资讯摘要提升到主内容区可见位置，作为「默认显示」，完整资讯保留在 Tab 内。

产品设计诊断

两类用户的行为模型与当前设计的适配缺口

基于用户行为数据，53% 的用户是「价格决策型」，24% 是「资讯研究型」。当前设计在视觉层级与信息路径上对这两类用户的支持存在明显落差。

53%
价格决策型 · Type A
看到合适价格，直接下单
用户路径
选择币对→ 看汇率→ 判断合适→ 立即下单

核心诉求：汇率数字足够大且醒目，CTA 在视野内始终可见且容易触达，路径尽可能短。
当前痛点：右侧操作面板视觉权重不足，与内容区缺乏明确边界，CTA 按钮不够显性。

24%

看新闻理解走势，再决策
用户路径
浏览资讯→ 理解走势→ 形成判断→ 决策下单

核心诉求：无需点击即可感知「有资讯可读」，资讯与汇率图表在同一视野内，读完资讯后 CTA 触手可及。
当前痛点：资讯藏在「市场资讯」Tab 内，不主动点击则完全不可见，用户可能不知道有资讯内容。

问题诊断 · 4 个关键缺口
Issue 01 · 高优先级

CTA 面板视觉权重不足

右侧「立即兑换」面板与主内容区用相同白底，无明显视觉分层。对于 Type A 用户，他们眼睛扫到汇率后，目光需要「找到」CTA，而不是「直接看见」它。操作入口应该是页面上视觉权重最高的功能区之一。

影响 Type A (53%)
Issue 02 · 高优先级

资讯对 Type B 用户完全不可见

市场资讯、财经日历等内容藏在 Sub-Tab 点击之后，默认状态下「概览」页面内没有任何资讯信号。Type B 用户不会因为不知道有资讯而主动点击，这意味着他们可能在没有充足信息支持的情况下就直接关闭页面。

影响 Type B (24%)
Issue 03 · 中优先级

汇率数字缺少「机会信号」上下文

大价格数字（6.9854）是绝对值，用户无法直接判断「这个价格好不好」。缺乏相对参照——比如「近 7 日分位」或「优于过去 X 天均值」——Type A 用户需要自己在脑中换算，增加了下单决策的认知摩擦。

影响 Type A (53%)影响 Type B (24%)
Issue 04 · 中优先级

顶部货币卡片的选中态与内容区视觉断层

卡片被选中后，下方内容区虽然联动更新，但两者之间没有视觉连接符（指示箭头 / 高亮延伸 / 卡片阴影投影），用户感知不到「当前展示内容来自于哪个卡片」，在切换多个货币对时容易产生方向感缺失。

优化计划

针对两类用户的分路径优化策略

核心原则：单一布局同时服务两种心智模型——Type A 路径要短且操作显性，Type B 路径要有被动信息曝光，且两条路径最终都汇聚到同一个高度可见的 CTA。

53%

Type A · 价格决策型 优化方向

1

加强 CTA 面板的视觉边界：右侧面板用浅暖色背景（#fff8f6）+ 品牌色左边框，与内容区形成明确的视觉分区，让用户眼睛扫视时第一时间感知到「这里是操作区」

2

增加汇率机会指示器：在汇率数字下方或 CTA 面板顶部，加入「近 7 日汇率分位条」（绿→红渐变 + 当前位置标记），让 Type A 用户在 1 秒内判断当前价格是否合适

3

CTA 按钮视觉升级：更大尺寸、更高对比度（渐变红 + 投影），并加入「下一步 ›」箭头，传递「点击即开始」的行动信号

4

货币卡片选中态优化：选中卡片增加品牌色底部三角连接符，建立卡片与下方内容区的视觉连接，消除空间断层感

24%

Type B · 资讯研究型 优化方向

1

在概览 Tab 中内置资讯预览条：图表下方、Sub-Tab 上方增加「最新资讯」预览区，默认展示 2-3 条资讯标题（不需点击 Tab），让 Type B 用户在进入页面时就感知到有资讯可读

2

资讯与图表自然衔接：预览条用淡黄色背景区分，视觉上像是图表的「文字注脚」，让用户从图表自然过渡到文字资讯，而不是靠点击 Tab 打断阅读流

3

Sub-Tab 上的资讯 Badge：「市场资讯」Tab 上显示红点/数字（如「2」），告知用户有未读资讯，降低用户「不知道有内容」的概率

4

资讯尾部嵌入 CTA：市场资讯 Tab 内容底部提供「根据当前分析，立即兑换」的内联 CTA，在 Type B 用户完成研究后，提供最短的转化路径
